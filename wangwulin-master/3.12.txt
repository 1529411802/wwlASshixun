git版本管理：
1、什么是版本管理：后悔，保存编辑的历史状态（和编辑菜单撤销功能类似）
2、git对项目文件夹的结构分割
工作区：编辑区
版本区（版本仓库）：位于项目文件夹下面子文件夹
	暂存区、历史区（成品区）
3、git进入
cd /d/dwh/项目文件夹
4、用户名初始化
git config  --global  user.name  ""
git config  --global  user.email  ""
5、项目文件夹版本区（仓库）初始化
git init
6、提交版本
1）将编辑区的修改提交到暂存区：
	git add *
2)将暂存区的内容提交到历史区：
	git commit -m  "版本信息"
7、查看历史版本
git log：只显示head指针以前的版本
git reflog：显示所有与版本有关的操作信息
8、head指针
含义：工作区是基于该指针指向的版本的内容

9、回撤
git reset --hard  版本号
作用：将历史版本的内容覆盖工作区的内容，将head指针指向该版本。但是不删除历史版本
10、版本前进
含义：将head指针以后版本的内容覆盖到工作区
git reset --hard  版本号


git的本地仓库：位于项目文件夹下面的子文件夹，存放暂存区和历史区（.git）
git的远程仓库：把本地仓库上传到网上，方便小组合作。但是和网盘有区别。
